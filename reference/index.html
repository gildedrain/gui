Page Layout

The following classes create a variety of page layouts against a 6-column grid, 960px wide, with 5px left & right margins, with 10px gutters between columns.

Keep in mind that each row can only add up to a maximum of 6.

class="first" forces the column to wrap to the next line. This prevents subsequent columns from filling whitespace to the right of the previous column.

class="precol#" adds left margin to the column

col# classes can be applied to almost any HTML tag

Important: When laying out a page using these classes, you need to lay out the entire page. You'll run into problems if you only try to layout one or two spots and let the rest of the text flow freely.

Clearfix & p.dots

.col#-grid-classes are positioned using a method called float. When you use CSS to position HTML elements and you invoke float, you effectively remove the floated element from the relative page flow. This becomes a problem when you have floating-children nested inside a parent-container. When you float a child element it's as if the child has moved out of the parent's house and the house shrinks to only fit the people it knows are living there. The examples below describe this scenario more visually. Adding class="clearfix" to the parent container will tell the parent-container to keep "containing" its floated children.

SyntaxResult

<div class="bg-medium-red">
  <h4>Without clearfix  (red = the parent-element)</h4>
  <div class="col1"><p>Child #1</p></div>
  <div class="col1"><p>Child #2</p></div>
</div>
<hr />
<div class="clearfix bg-medium-red">
  <h4>With clearfix</h4>
  <div class="col1"><p>Child #1</p></div>
  <div class="col1"><p>Child #2</p></div>
</div>

Another problem may occur when certain elements refuse to wrap to the next line. To force a hard line-break after previously using floated elements, you can add a paragraph with class="dots" like so. p.dots is 1px tall, has zero margins, zero padding, zero line-height and only contains a non-breaking-space character. It's effectively invisible.

SyntaxResult

Without p.dots
<div class="col1">Floated col1</div>
<p>This paragraph should start on a new line.</p>

With p.dots
<div class="col1">Floated col1</div>
<p class="dots">&nbsp;</p>

Horzy

Horzy is a javascript-driven horizontal page sliding effect. It stacks "slides" or "pages" horizontally and when you click the horzy navigation links it animates a horizontal scroll to the designated "slide/page". You can have multiple instances of Horzy active on a single page.

Verty

Verty is a javascript-driven vertical page sliding effect. When you click the verty navigation links it animates a vertical scroll to the designated anchor-link. The vertical-navigation is cloned and given a fixed position in the browser window so that it follows you as you scroll up and down the page.

Verty is currently being used to drive this reference guide. You can still click the demo links, but the effect isn't going to work 100%. You can only have one instance of Verty active on a single page.

While possible to use Verty inside a Content Block, I believe Verty is best used in the Collgrid area of the template file, allowing you to have maximum control over page layout, transparent backgrounds, full-bleed sections, etc.

BG_Slideshow

BG_Slideshow is a JQuery script that creates a very large background-graphic slideshow. It creates the slides for the slideshow from HTML elsewhere on the page. By default this HTML is hidden from view and just serves as a simple data-source.

Vertical placement of captions is accomplished by using caption-top, caption-middle, and caption-bottom classes. Horizontal placement of captions is accomplished by a combination of precol and col grid classes.

The slideshow navigation doesn't work 100% correctly in this reference guide. I honestly didn't expect it to work inside of a nested container. Neat!

BG_Banners

BG_Banners is a JQuery script that creates a mosaic of block-level hyperlinks. If they contain an image, the image will be applied as a background-graphic. A variety of GUI classes can be applied to each link: col classes for width, bg-color, text-color, font-typefaces, etc.

A future update will remove margin-top:-150px; as seen on www.sunyocc.edu/home.aspx. Margin-top should be 0 universally, and exceptions should be applied in a local stylesheet.
